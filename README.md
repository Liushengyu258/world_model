# 简易世界模型 (Simple World Model)

本项目是一个为 Mac (Apple Silicon 16GB 内存) 设计的极简版“世界模型”。用于帮助初学者理解世界模型的基本运作原理：**给定当前世界的状态，预测世界未来的走向**。

## 任务背景：Moving MNIST
为了在个人电脑上能快速跑通流程，我们选择了一个经典且轻量的任务：**Moving MNIST**。
该任务包含两个在 64x64 空间中做匀速直线运动、碰到边缘会反弹的手写数字。模型需要通过观察视频，学会其中的“物理运动规律”。

## 模型架构
真正的环境太过复杂，所以世界模型通常不会直接去预测下一张高清图片，而是预测“特征”。本项目包含两个轻量的神经网络：

1. **感知模块 (Autoencoder, `model.py`)**
   - **作用**：将高维复杂的图像压缩成低维的核心特征向量（128维），并负责在最后将特征重新解码成画面。相当于给世界降维。
   - **注意**：代码中使用了 `BCELoss` (二元交叉熵) 损失函数，强迫模型更快地学习黑白图像中的白色数字特征，避免它偷懒输出全黑画面。
2. **动力学模块 (LSTM, `model.py`)**
   - **作用**：接收过去的特征序列，推演并预测下一个时刻的特征向量。这相当于在抽象的隐空间里模拟未来的物理规律。

## 运行指南

本项目环境和代码已经搭建完毕，请按照以下步骤运行。

### 1. 激活虚拟环境
在项目根目录下打开终端，激活环境：
```bash
source venv/bin/activate
```

### 2. 训练模型 (Train)
运行训练脚本。脚本会自动下载数据集（约 160MB，第一次运行需要稍等），并自动开启 Mac 的 `mps` 硬件加速。
```bash
python -u train.py
```
> **训练策略**：模型分阶段训练。
> - 第一阶段 (Phase 1)：仅训练自编码器，让模型学会把图像压缩再还原。
> - 第二阶段 (Phase 2)：冻结自编码器，专门训练 LSTM，让它学会预测下一个特征。
> 
> **⚠️ 重新训练的重要提示**：代码中包含自动跳过机制。如果你想修改训练轮数重新训练，**必须先手动删除项目下的 `ae_weights.pth` 和 `dyn_weights.pth` 文件**，否则程序会直接加载旧权重并结束。

### 3. 推理与可视化 (Inference)
当训练完成并生成了权重文件后，你可以让模型开始“做梦”（推演未来）：
```bash
python inference.py
```
模型会随机抓取一段测试视频，**只看前 5 帧**作为“引子”，然后自回归地**推演预测后面的 15 帧**。最终结果会自动保存为 `world_model_dream.gif` 文件。

### 📊 如何解读生成的 GIF？
打开生成的 GIF，你会看到左右对比的画面：
- **左边 (Ground Truth)**：真实的未来画面，数字严格按照物理规律移动和反弹。
- **右边 (Model Dream)**：模型脑补的未来。右侧的前 5 帧是提供给模型的“引子”（和左边完全一致）。从第 6 帧开始，是模型纯靠自己推演出的未来。

**为什么初始生成的画面很模糊、甚至是一团马赛克？**
为了让你能在几分钟内跑通基本流程，我们默认的训练轮数非常少（比如 2 轮）。这就导致 Autoencoder 还没能完美还原数字的形状，而 LSTM 预测时产生的一点点误差，在自回归推演中（即用自己生成的第6帧去预测第7帧）像滚雪球一样迅速放大，最后解码出来就是一团乱码。这对于早期的连续潜空间模型是完全正常的现象。

## 进阶尝试 (让模型变得真正清晰)
如果你想看到模型真正“学会”物理规律，预测出清晰且轨迹准确的未来，你需要大幅增加训练时间：
1. 删除旧的权重文件：`rm ae_weights.pth dyn_weights.pth`
2. 打开 `train.py`，将 `ae_epochs` 改为 `20`，将 `dyn_epochs` 改为 `50` (或更高)。
3. 重新运行 `python -u train.py`。
4. 这个过程可能在 Mac 上需要运行半小时左右（如果是高端的 NVIDIA 显卡会快非常多）。训练越久，右边模型脑补的未来就会越锐利，轨迹也就越准！
